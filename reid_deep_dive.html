1 The ReID Problem

What is Re-Identification?

Re-Identification (ReID) is the task of matching a query image of a person or vehicle against a gallery of images captured by different cameras, at different times, and under different conditions — without relying on persistent identifiers like license plates or face images.

This is fundamentally different from classification: in ReID the model must generalize to identities never seen during training. The correct framing is metric learning — learning a distance function such that images of the same identity are closer than images of different identities in the embedding space.

Key distinction vs. VMMR/classification: Classification is closed-set (196 fixed classes). ReID is open-set: at test time, the model sees query identities it has never trained on. The model must produce a ranking of the gallery, not a class label.

Query-Gallery Paradigm

Person ReID vs. Vehicle ReID

Person ReID

  • Discriminative cues: clothing color, texture, accessories, gait, body shape
  • High viewpoint variability (front/back/side look very different)
  • Occlusion is frequent (crowds, objects)
  • Clothing changes possible across days (long-term ReID)
  • Part alignment critical: head, torso, legs carry different information
  • Typical image size: 64×128 or 128×256 pixels

Vehicle ReID

  • Discriminative cues: color, shape, minor part differences (headlights, wheels, stickers)
  • Hard negatives: same make/model/color but different instance (identical twins problem)
  • Viewpoint matters enormously: front vs. rear vs. side are near-orthogonal appearances
  • Spatial-temporal context (trajectory, camera topology) is highly informative
  • License plate is a strong cue but may be occluded, blurry, or deliberately absent
  • Typical image size: 128×128 or 224×224 pixels

3 Core Challenges

Viewpoint & Pose Variation

The same person/vehicle looks radically different from front, side, and rear. A vehicle's front and rear views share almost no visual features yet must be matched. Part-based models and attention mechanisms address this.

Illumination & Weather

Outdoor cameras span dawn to night, sunny to overcast. Color histograms shift drastically. Models must learn illumination-invariant features or rely on texture/shape over color.

Resolution & Quality

Surveillance images are often low-resolution (64×128 for person ReID). Super-resolution preprocessing or multi-scale feature extraction helps, but fundamentally limits what discriminative detail is available.

Occlusion

Partial visibility is common — a person half-hidden behind a pillar, a vehicle with its rear occluded. Occluded ReID is a distinct research area requiring part visibility estimation and spatial attention.

The Identical Twin Problem (Vehicle)

Two vehicles of the same make, model, color, and year are visually near-identical. Distinguishing them requires subtle cues: minor dents, stickers, custom accessories. Spatial-temporal cues (camera network topology) provide critical disambiguation.

Domain Gap

Models trained on Market-1501 (outdoor university) often degrade on MSMT17 (multi-environment). Unsupervised domain adaptation (UDA) and domain-generalized ReID are active research directions.

Long-term ReID (person): When the same person is observed days or weeks apart, they may have changed clothing entirely. Clothing-agnostic features — body shape, gait, face, tattoos — become the only reliable cues. This is an emerging and unsolved challenge.

Stride trick for ReID: Setting the last residual block's stride from 2 to 1 (in ResNet-50) doubles the spatial resolution of the feature map from 8×16 to 16×32 (for 256×128 input). This is critical for part-based models and horizontal stripe pooling because it preserves the spatial layout needed to extract head/torso/leg features separately.
OSNet: Omni-Scale Network architecture details

OSNet (Zhou et al., 2019) was designed specifically for person ReID with three design goals:

  • Omni-scale feature learning: Each block has multiple receptive field sizes (depthwise conv of varying dilation/kernel) whose outputs are aggregated via a learned Unified Aggregation Gate (AG). This captures both fine-grained texture and coarse body shape.
  • Lightweight: Only 2.2M parameters (10× smaller than ResNet-50) via depthwise-separable convolutions — suitable for real-time inference.
  • Pointwise aggregation: Feature maps from different scales are fused element-wise after the AG, not by concatenation, keeping feature dimension constant.

Despite its tiny size, OSNet-AIN (with Instance Normalization layers for appearance invariance) achieves competitive results on Market-1501 (~94% Rank-1) — often better than ResNet-50-based methods.

TransReID: Vision Transformer for ReID

TransReID (He et al., 2021) adapts ViT for person and vehicle ReID with two key innovations:

  • Jigsaw Patch Module (JPM): Randomly shuffles patch groups and feeds them to separate transformer branches, creating diverse part-level features without requiring explicit part detection. This encourages the model to learn robust local representations from individual spatial regions.
  • Side Information Embeddings (SIE): Camera ID and viewpoint labels are encoded as learnable embeddings and added to the patch tokens. This allows the model to explicitly condition its feature extraction on the known camera/viewpoint, greatly reducing camera-induced style variation in the embedding space.

TransReID-SSL (using DINOv2 or MoCo pretraining instead of supervised ImageNet) further improves performance by 2–4% mAP on MSMT17.

BN-Neck: The Standard ReID Trick

The BN-Neck (Luo et al., 2019 — "Bag of Tricks") is the single most impactful technique in modern ReID training. The key insight is to use different features for the two training objectives:

class ReIDModel(nn.Module):
    def __init__(self, num_ids, embed_dim=256):
        super().__init__()
        backbone = resnet50(pretrained=True)
        # Stride trick: last block stride 2 → 1
        backbone.layer4[0].downsample[0].stride = (1, 1)
        backbone.layer4[0].conv2.stride          = (1, 1)
        self.backbone = nn.Sequential(*list(backbone.children())[:-2])

        self.pool    = nn.AdaptiveAvgPool2d(1)   # GAP → (B, 2048)
        self.bn_neck = nn.BatchNorm1d(2048)      # BN-Neck (no bias)
        self.bn_neck.bias.requires_grad_(False)

        # Classifier uses BN features (better ID loss convergence)
        self.classifier = nn.Linear(2048, num_ids, bias=False)

        # Embedding head uses pre-BN or projected features for triplet loss
        self.embed_proj = nn.Sequential(
            nn.Linear(2048, embed_dim, bias=False),
            nn.BatchNorm1d(embed_dim)
        )

    def forward(self, x):
        feat   = self.pool(self.backbone(x)).flatten(1)   # (B, 2048) raw
        neck   = self.bn_neck(feat)                         # BN-normalized
        logits = self.classifier(neck)                      # ID classification
        emb    = F.normalize(self.embed_proj(neck), p=2, dim=1)  # L2 embed
        return logits, emb, feat  # feat used for triplet at inference-time search
Inference note: At test time, use the BN-normalized features (neck) or the projected embedding (emb) for distance computation — not the raw pre-BN features. The BN normalization removes camera/domain style variation, making embeddings from different cameras more comparable.

6 Dual-Head Architecture for ReID

The standard modern ReID architecture combines an ID classification head (cross-entropy / label smoothing) with an embedding head (triplet loss) over a shared backbone. This joint training is often called the "ID loss + triplet loss" paradigm and forms the backbone of the Bag-of-Tricks (BOT) baseline and most SOTA methods.

Why ID Loss + Triplet Loss?

ID Loss (Cross-Entropy) role

  • Provides strong, dense gradient signal from the first epoch
  • Forces inter-class separation in the classifier weight space
  • Stabilizes early training — triplet loss alone often diverges initially
  • The learned classifier weights become class prototypes useful for zero-shot retrieval

Triplet Loss role

  • Directly optimizes the embedding space geometry (what we use at test time)
  • Enforces a margin between same-ID and different-ID distances
  • Produces compact intra-class clusters and well-separated inter-class clusters
  • More aligned with the Rank-1 / mAP retrieval objective than CE

Combined Loss Formula

ReID Combined Loss (BOT formulation)
L_total = L_ID(logits, y, ε=0.1) + L_triplet(embeddings, y, m=0.3)

Note: equal weighting (α=β=1.0) works well; some works use α=1.0, β=0.3–0.5 early in training

Variant Options

Option A: ID + Triplet (recommended)

Standard BOT baseline. Triplet uses batch-hard mining with PKSampler (P=16, K=4).

★ Best starting point

Option B: ID + ArcFace

7 Metric Learning for ReID

7.1 Triplet Loss (Batch-Hard Mining)

Batch-Hard Triplet Loss (Hermans et al., 2017)
L = (1/P·K) Σi max(0, maxj: y_j=y_i d(eᵢ,eⱼ) − mink: y_k≠y_i d(eᵢ,eₖ) + m )

d = Euclidean distance on L2-normalized embeddings, m = margin (0.3 typical) 
def batch_hard_triplet_loss(emb, labels, margin=0.3):
    # emb: (N, D) L2-normalized; labels: (N,)
    dist  = 2 - 2 * (emb @ emb.T)          # sq-Euclidean on unit sphere
    dist  = dist.clamp(min=0).sqrt()
    same  = labels.unsqueeze(0) == labels.unsqueeze(1)
    pos_d = (dist * same.float()).max(dim=1).values          # hardest positive
    neg_d = dist.masked_fill(same, 1e9).min(dim=1).values    # hardest negative
    return (pos_d - neg_d + margin).clamp(min=0).mean()

7.2 Circle Loss

Circle Loss (Sun et al., 2020) unifies pair-based and class-based metrics by assigning individual weights to each similarity score, allowing the model to focus differently on similarities that are far from the decision boundary. It subsumes both contrastive and triplet losses as special cases.

Circle Loss
L = log[ 1 + Σj∈neg exp(γ·α_n_j·(s_n_j − Δ_n)) · Σi∈pos exp(−γ·α_p_i·(s_p_i − Δ_p)) ]

where s = cosine similarity, α = max(0, O±s) is the per-sample weighting factor,
O_p=1+m, O_n=−m, Δ_p=1−m, Δ_n=m, γ=scale
Circle Loss in practice: Use m=0.25, γ=128. Drop-in replacement for triplet loss. Often outperforms it by 1–3% mAP on MSMT17 without requiring PKSampler or manual mining strategy changes.

7.3 ArcFace / CosFace for ReID

Adapted from face recognition, ArcFace replaces the standard cross-entropy ID loss with an angular margin version. Requires L2-normalized embeddings and weight vectors.

ArcFace ID Loss for ReID
L_arc = CE(s·[cos(θ_y+m), cos(θ_j≠y)], y)
where θ_y = arccos(W_y^T · e), s=30 (scale), m=0.5 (margin) 
class ArcFaceHead(nn.Module):
    def __init__(self, embed_dim, num_ids, s=30.0, m=0.50):
        super().__init__()
        self.s, self.m = s, m
        self.W = nn.Parameter(torch.empty(num_ids, embed_dim))
        nn.init.xavier_uniform_(self.W)

    def forward(self, emb, labels):
        cos_theta = F.linear(F.normalize(emb), F.normalize(self.W))
        theta      = cos_theta.acos().clamp(0, math.pi)
        target_cos = (theta + self.m).cos()
        logits     = cos_theta.scatter(1, labels.unsqueeze(1), target_cos.unsqueeze(1))
        return F.cross_entropy(self.s * logits, labels)

7.4 Center Loss

Center Loss (Wen et al., 2016) maintains a learnable centroid for each class and penalizes the distance from each embedding to its class center, reducing intra-class variance:

Center Loss
L_center = (1/2) Σi ‖eᵢ − c_{yᵢ}‖²
L_total = L_ID + λ · L_center + L_triplet (λ=0.0005 typical)

Particularly effective for vehicle ReID where the same vehicle instance must be matched despite large viewpoint changes, and the class centroids naturally converge to a viewpoint-invariant representation.

8 Part-Based Models

Global pooling discards spatial structure. Part-based models explicitly decompose the image into spatial regions (stripes, semantic parts) and extract separate embeddings for each region, then concatenate or aggregate them. This is particularly powerful for person ReID where head, torso, and legs carry complementary information.

PCB: Part-Based Convolutional Baseline

PCB (Sun et al., 2018) divides the feature map into K horizontal stripes and applies separate classifiers to each stripe's average-pooled representation.

MGN: Multiple Granularity Network

MGN (Wang et al., 2018) extends PCB with three parallel branches of different granularities:

  • Branch 1: Global feature (no stripe split) — captures holistic identity
  • Branch 2: 2-stripe split — upper/lower body
  • Branch 3: 3-stripe split — head/torso/legs

Each branch has its own classifier and contributes to the total ID loss. At inference, all branch embeddings are concatenated to form the final descriptor. MGN achieves ~95% Rank-1 on Market-1501 and is still competitive with transformer-based methods.

AlignedReID

AlignedReID (Zhang et al., 2017) uses a dynamic alignment module that learns to find the best horizontal alignment between two person images before computing part-level distances, addressing misalignment due to pose variation.

9 Training Strategy

PKSampler for ReID

ReID training requires a PK-sampler (P identities × K images each per mini-batch) to guarantee valid triplets. Typical settings: P=16, K=4 → batch=64.

class PKSampler(Sampler):
    def __init__(self, dataset_labels, P=16, K=4):
        self.P, self.K = P, K
        self.pid_index = defaultdict(list)
        for i, pid in enumerate(dataset_labels):
            self.pid_index[pid].append(i)
        self.pids = [p for p, idxs in self.pid_index.items() if len(idxs) >= K]

    def __iter__(self):
        random.shuffle(self.pids)
        for i in range(0, len(self.pids) - self.P + 1, self.P):
            batch_pids = self.pids[i:i+self.P]
            batch_idxs = []
            for pid in batch_pids:
                idxs = random.choices(self.pid_index[pid], k=self.K)
                batch_idxs.extend(idxs)
            yield from batch_idxs

Bag-of-Tricks (BOT) Full Training Recipe

The "Bag of Tricks for Person Re-identification" (Luo et al., 2019) standardized modern ReID training. Its components, each contributing incrementally:

Training Schedule

Optimizer & LR

  • Optimizer: SGD (momentum=0.9) or AdamW
  • Base LR: 0.035 (SGD) or 3e-4 (Adam)
  • Warmup: 10 epochs linear ramp
  • Decay: cosine (or step at epoch 40, 70)
  • Total epochs: 80–120

Batch & Sampler

  • PKSampler: P=16, K=4 → batch=64
  • Weight decay: 5e-4
  • Gradient clip: max_norm=1.0
  • Mixed precision (fp16) for speed
  • Early stopping on mAP

Camera-Aware Training (Vehicle ReID)

For vehicle ReID, explicitly conditioning on camera ID improves results. Three strategies:

  • Camera ID embedding (SIE): Add a learnable camera embedding to the CLS token (TransReID)
  • Camera-aware normalization: Separate BN statistics per camera to remove camera-induced style shift
  • Hard negative mining across cameras: In PKSampler, ensure negatives are sampled from different cameras to avoid camera-correlated shortcuts

10 Evaluation

Evaluation Protocol

Standard ReID evaluation:

  1. Extract embeddings for all query and gallery images
  2. For each query, compute distance to every gallery image
  3. Sort gallery by ascending distance (rank list)
  4. Compute CMC and mAP over all queries
Same-camera same-ID images are excluded from the gallery during evaluation — a query from camera 1 does not evaluate against gallery images from camera 1 of the same person, since the same camera is trivially easy to match.

CMC Curve (Cumulative Matching Characteristics)

CMC@K = probability that the correct match appears within the top-K ranked results. Rank-1 (CMC@1) is the most reported metric.

CMC@K
CMC(K) = (1/|Q|) Σq 𝟙[rank(correct match for q) ≤ K]

Mean Average Precision (mAP)

mAP accounts for multiple correct matches in the gallery (same ID across multiple cameras). It penalizes models that retrieve correct matches at low ranks.

mAP
AP(q) = (1/|R_q|) Σk=1|G| P(k) · rel(k)

mAP = (1/|Q|) Σq AP(q)

|R_q| = number of relevant gallery images for query q; P(k) = precision at rank k; rel(k) = 1 if rank k is relevant 
def compute_map_cmc(qf, gf, q_pids, g_pids, q_cams, g_cams):
    # qf: (Nq,D) query embeddings; gf: (Ng,D) gallery embeddings
    dist = 1 - qf @ gf.T              # cosine distance (L2-normalized)
    cmc_scores, ap_scores = [], []
    for qi in range(len(qf)):
        order  = dist[qi].argsort()
        # Remove same-camera same-id distractors
        valid  = ~((g_pids[order] == q_pids[qi]) & (g_cams[order] == q_cams[qi]))
        ranked = g_pids[order][valid]
        # CMC
        hits = (ranked == q_pids[qi]).astype(float)
        cmc_scores.append(hits.cumsum())
        # AP
        idx  = np.where(hits)[0] + 1
        ap_scores.append((np.arange(1, len(idx)+1) / idx).mean() if len(idx) else 0)
    cmc = np.mean(cmc_scores, axis=0)
    return cmc[0], np.mean(ap_scores)  # Rank-1, mAP

mINP (Mean Inverse Negative Penalty)

mINP (Ye et al., 2021) measures the cost of finding all correct matches — it focuses on the hardest positive (the correct match at the highest rank). More sensitive than mAP for large-gallery evaluation:

mINP
INP(q) = |R_q| / R_max(q) where R_max is the rank of the last relevant item
mINP = (1/|Q|) Σq INP(q)
