Vehicle Make & Model
Recognition (VMMR)

A comprehensive study of fine-grained vehicle recognition — covering backbone architectures, embedding generation, metric learning approaches, and the dual-head training paradigm using the Stanford Cars dataset.

1 Introduction to VMMR

What is Fine-Grained Visual Recognition?

Coarse classification distinguishes between fundamentally different visual categories (car vs. truck vs. bus). Fine-grained visual recognition (FGVR) operates within a single basic-level category — distinguishing a 2011 BMW 3 Series Sedan from a 2012 BMW 3 Series Sedan, or a Toyota Camry from a Toyota Corolla.

VMMR is an instance of FGVR applied to vehicles. The output is typically a joint label (Make, Model, Year), which maps to one of the N leaf classes in a taxonomy. For Stanford Cars, N = 196.

Key distinction: In VMMR the inter-class visual distance can be smaller than intra-class distance. Two instances of the same model photographed from different angles can look more dissimilar than two different models photographed identically. This is what makes VMMR fundamentally harder than standard classification.

Why VMMR is Challenging

Inter-class Similarity

Consecutive model years of the same vehicle often differ only in minor detail — grill shape, badge position, wheel design. Subtle cues invisible to humans at scale must be learned.

Intra-class Variation

The same vehicle model can appear vastly different due to viewpoint, lighting, occlusion, color, after-market modification, and partial capture in surveillance footage.

Long-tail Distribution

In real-world datasets, rare models (exotic sports cars, limited editions) have far fewer training examples than common models, causing class imbalance.

Part Localization

Discriminative regions (headlights, grill, badge) are small relative to the full image. Models must implicitly localize these regions without explicit part supervision.

Applications

  • Traffic surveillance & law enforcement: Identify stolen or suspect vehicles from CCTV
  • Insurance & claims processing: Verify reported vehicle make/model automatically
  • Autonomous driving: Understand surrounding vehicle types for behavioral prediction
  • Fleet management: Automated inventory and tracking by model
  • Parking management: Permit validation by registered vehicle model
  • Automotive retail: Visual search ("find cars similar to this photo")

2 Backbone Architectures

The backbone is a deep convolutional (or transformer-based) network that maps a raw image tensor to a compact feature vector. For VMMR we almost always initialize from ImageNet-pretrained weights and fine-tune — because 8,144 training images are insufficient to learn good low-level visual features from scratch.

Transfer Learning Rationale

Low-level features (edges, textures, colors) and mid-level features (object parts, surfaces) learned on ImageNet transfer effectively to vehicle images. Only the high-level semantic features and the final decision boundary need to be adapted. Fine-tuning typically yields 15–25% absolute accuracy improvement over training from scratch on small datasets like Stanford Cars.

Deep dive: EfficientNet compound scaling

EfficientNet (Tan & Le, 2019) proposes compound scaling: simultaneously scaling network depth, width, and input resolution using a fixed ratio φ:

Scaling Equations
depth: d = αφ, width: w = βφ, resolution: r = γφ
subject to: α · β² · γ² ≈ 2, α ≥ 1, β ≥ 1, γ ≥ 1

This ensures all three dimensions are scaled proportionally, preventing the common failure mode of only scaling one dimension (e.g., very deep but narrow networks). EfficientNet-B0 is the baseline; B1-B7 multiply φ. This explains why all EfficientNet variants share the same architectural skeleton but differ in scale.

Deep dive: Vision Transformer (ViT) for fine-grained recognition

ViT (Dosovitskiy et al., 2020) splits the image into fixed 16×16 (or 14×14) patches, linearly embeds each patch, and processes them with a standard Transformer encoder. The CLS token's output at the last layer is used as the global image representation.

For FGVR, ViT's global attention is particularly powerful: the model can directly attend to discriminative regions (e.g., headlight shape, grill pattern) across the entire image without the locality constraint of convolutions. DINOv2-pretrained ViT features exhibit strong semantic structure even without task-specific fine-tuning, making them especially well-suited as embedding backbones.

Caveat: ViT requires more images or longer training to converge well when fine-tuning from scratch. Using pretrained DINOv2 or CLIP weights mitigates this.

3 Embedding Generation

An embedding is a dense, low-dimensional vector representation of an image such that semantically similar images map to nearby points in the embedding space. The geometry of this space is what metric learning methods directly optimize.

From Backbone to Feature Vector

Most CNN backbones end with a sequence of convolutional blocks producing a spatial feature map of shape (B, C, H, W). Converting this to a fixed-size vector is done via:

Global Average Pooling (GAP)

Average each spatial location over the channel dimension. Produces a (B, C) vector. Standard in ResNet, EfficientNet.

fi = (1/HW) Σh,w xi,h,w

Retains semantic content; discards spatial position.

Global Max Pooling (GMP)

Take the maximum activation per channel. More discriminative for detecting the presence of a feature anywhere in the image, but can be noisier.

fi = maxh,w xi,h,w

Often combined with GAP in fine-grained tasks.

Projection Head Design

The projection head maps the backbone feature vector (e.g., 1280-d from EfficientNet-B0) to a lower-dimensional embedding space used for metric learning. A common design: 

class EmbeddingHead(nn.Module):
    def __init__(self, in_features=1280, embed_dim=256):
        super().__init__()
        self.proj = nn.Sequential(
            nn.Linear(in_features, embed_dim, bias=False),
            nn.BatchNorm1d(embed_dim),          # stabilizes triplet training
        )

    def forward(self, x):
        emb = self.proj(x)
        return F.normalize(emb, p=2, dim=1)  # unit hypersphere

Why L2 Normalization Matters

Projecting embeddings onto the unit hypersphere (‖e‖₂ = 1) has two critical effects:

  • Bounded distances: Euclidean distance between unit vectors is in [0, 2], making triplet margins interpretable and training more stable.
  • Cosine similarity as dot product: For unit vectors, cosine similarity equals the dot product — enabling efficient similarity search with FAISS or ScaNN at inference time.
  • Prevents embedding collapse: Without normalization, the network can minimize triplet loss by shrinking all embeddings toward zero. L2 normalization eliminates this degenerate solution.

The BN-Neck Trick

BN-Neck (Luo et al., 2019) inserts a BatchNorm layer between the backbone features and the classification/embedding heads. The key insight: the classification head sees the BatchNorm-normalized features (which have better gradient flow for CE loss), while the embedding head optionally operates before normalization. This decouples the two learning objectives partially, stabilizing joint training with combined loss.

4 Metric Learning Approaches

Metric learning trains the model so that the learned distance function d(x, y) reflects semantic similarity: small distance for same-class pairs, large distance for different-class pairs. Below are the five major families, with deep dives on each.

4.1 Contrastive Loss (Siamese Networks)

Operates on pairs of images (x₁, x₂) with a binary label y ∈ {0, 1} indicating same or different class.

Contrastive Loss (Hadsell et al., 2006)
L = y · d(e₁, e₂)² + (1 − y) · max(0, m − d(e₁, e₂))²

For same-class pairs (y=1), the loss minimizes distance. For different-class pairs (y=0), it penalizes if the distance falls below margin m.

Pros

  • Simple, well-understood
  • Easy to implement
  • Natural for verification tasks

Cons

  • Requires careful pair construction
  • O(N²) pairs — hard to mine efficiently
  • No relative ordering (only absolute threshold)

4.2 Triplet Loss

Operates on triplets (anchor a, positive p, negative n): same class for (a,p), different class for (a,n). The objective enforces d(a,p) + margin < d(a,n).

Triplet Loss (Schroff et al., FaceNet 2015)
L = Σi max(0, d(aᵢ, pᵢ)² − d(aᵢ, nᵢ)² + α)

Mining Strategies

The choice of which triplets to present during training is critical. Easy triplets (already satisfying the margin) provide zero gradient. Hard triplets are the most informative but can cause instability early in training.

StrategyPositive selectionNegative selectionStabilitySpeed
Batch Random Random positive Random negative High Fast
Semi-Hard Random positive Neg: d(a,n) > d(a,p), but d(a,n) < d(a,p) + margin Medium Medium
Batch Hard Hardest positive (max d in batch) Hardest negative (min d in batch) Medium Fast
Batch All All positives All valid negatives Low (early training) Slow
Batch-hard mining: algorithm walkthrough

Given a batch of N embeddings with labels, compute the full N×N distance matrix. For each anchor i:

  • Hardest positive: Among all j with label[j] == label[i], pick j* = argmax d(i, j)
  • Hardest negative: Among all k with label[k] ≠ label[i], pick k* = argmin d(i, k)
def batch_hard_triplet_loss(embeddings, labels, margin=0.3):
    # embeddings: (N, D), L2-normalized
    dist = 2 - 2 * embeddings @ embeddings.T    # squared Euclidean on unit sphere
    dist = dist.clamp(min=0).sqrt()

    same = labels.unsqueeze(0) == labels.unsqueeze(1)  # (N, N) bool

    # Hardest positive: max dist among same-class pairs
    pos_dist = (dist * same.float()).max(dim=1).values

    # Hardest negative: min dist among diff-class pairs
    neg_dist = dist.masked_fill(same, float('inf')).min(dim=1).values

    loss = (pos_dist - neg_dist + margin).clamp(min=0)
    return loss.mean()

4.3 ArcFace / CosFace (Angular Margin Losses)

Instead of operating on pairs or triplets, ArcFace/CosFace work within the classification framework but add an angular margin to the softmax logits, directly optimizing the geometry of the embedding space.

ArcFace Loss (Deng et al., 2019)
L = −log [ e^(s · cos(θ_yᵢ + m)) / (e^(s · cos(θ_yᵢ + m)) + Σj≠yᵢ e^(s · cos(θⱼ))) ]
where θ_yᵢ = angle between embedding and class weight vector, s = scale, m = angular margin
CosFace (Wang et al., 2018)
L = −log [ e^(s · (cos(θ_yᵢ) − m)) / (e^(s · (cos(θ_yᵢ) − m)) + Σj≠yᵢ e^(s · cos(θⱼ))) ]

ArcFace adds the margin m to the angle before computing cosine — this creates a stricter angular boundary than CosFace's additive cosine margin, but both achieve state-of-the-art results on face verification and vehicle re-identification benchmarks.

Why this is powerful for VMMR: ArcFace forces the network to separate classes by a minimum angular gap on the hypersphere. The class weight vectors become well-spaced "class prototypes." At inference, classification and embedding-based retrieval both work from the same learned geometry.

4.4 ProxyNCA

ProxyNCA (Movshovitz-Attias et al., 2017) assigns a learnable proxy vector to each class. Instead of mining pairs/triplets of images, each image is compared only against these proxies — dramatically reducing the O(N³) problem of triplet mining to O(N).

ProxyNCA Loss
L(x) = −log [ exp(−d(x, p_{y})) / Σz≠y exp(−d(x, p_z)) ]

Proxies are trainable parameters and converge to be representative of their class cluster. No mining required; loss is stable and efficient. Particularly well-suited when classes are well-separated (which is not always the case in VMMR).

4.5 Supervised Contrastive Loss (SupCon)

SupCon (Khosla et al., NeurIPS 2020) extends self-supervised contrastive learning (SimCLR) to the supervised setting: for each anchor, all same-class images in the batch serve as positives, and all different-class images serve as negatives.

SupCon Loss
L = Σi (−1/|P(i)|) Σp∈P(i) log [ exp(zᵢ·zₚ/τ) / Σa∈A(i) exp(zᵢ·zₐ/τ) ]
where P(i) = same-class indices, A(i) = all other indices, τ = temperature

SupCon is significantly more stable than triplet loss (no fragile mining needed) and often outperforms it. It naturally makes use of all positive pairs in the batch, giving richer gradient signal. Temperature τ (typically 0.07) controls the concentration of the distribution.

5 Dual-Head Architecture

The dual-head architecture trains a single backbone jointly under two complementary objectives: a classification head optimized with cross-entropy, and an embedding head optimized with triplet loss. The backbone is shared — gradients from both losses propagate through it simultaneously.

Motivation for the Dual-Head Design

  • Cross-entropy alone produces a classification boundary but may not structure the feature space for retrieval. Similar classes may share a region of the embedding space but aren't explicitly separated.
  • Triplet loss alone can produce a well-structured embedding space but the training is fragile — requires careful mining and the PKSampler; convergence is slower.
  • Combined: CE provides stable, fast early convergence (class signal is clear from the beginning). Triplet loss then refines the embedding geometry. The two objectives complement each other.

Architecture Diagram

Combined Loss

Total Training Loss
L_total = α · L_CE(logits, y, ε=0.1) + β · L_triplet(embeddings, y, m=0.3)

where α, β are loss weights (default: α=1.0, β=1.0) 
class CombinedLoss(nn.Module):
    def __init__(self, alpha=1.0, beta=1.0, margin=0.3, label_smoothing=0.1):
        super().__init__()
        self.alpha = alpha
        self.beta = beta
        self.ce = nn.CrossEntropyLoss(label_smoothing=label_smoothing)
        self.margin = margin

    def forward(self, logits, embeddings, labels):
        l_ce = self.ce(logits, labels)
        l_tri = batch_hard_triplet_loss(embeddings, labels, self.margin)
        return self.alpha * l_ce + self.beta * l_tri, l_ce.item(), l_tri.item()

Alternative Dual-Head Options

Option A — Shared BN-Neck (Recommended)

Both heads branch from a single shared BatchNorm neck after GAP. Heads share the normalized representation.

★ Recommended — most stable joint training

Option B — Independent Heads

Both heads attach directly to the raw GAP output with no shared neck. Each head has its own BN/projection.

Moderate — slightly more flexible, more params

Option C — Triple Head (CE + Triplet + ArcFace)

Adds an ArcFace head alongside CE and triplet. Maximum discriminativeness but sensitive to hyperparameters.

Advanced — use after baseline is working
Model code skeleton (PyTorch) 
import torch, torch.nn as nn, torch.nn.functional as F
import torchvision.models as models

class VMMRModel(nn.Module):
    def __init__(self, num_classes=196, embed_dim=256, dropout=0.3):
        super().__init__()
        # Backbone: EfficientNet-B0, remove classifier
        backbone = models.efficientnet_b0(weights="IMAGENET1K_V1")
        self.features = backbone.features       # conv blocks
        self.pool     = nn.AdaptiveAvgPool2d(1)  # GAP
        feat_dim      = backbone.classifier[1].in_features  # 1280

        # BN-Neck
        self.bn_neck = nn.BatchNorm1d(feat_dim)
        self.dropout = nn.Dropout(p=dropout)

        # Embedding head
        self.embed_head = nn.Sequential(
            nn.Linear(feat_dim, embed_dim, bias=False),
            nn.BatchNorm1d(embed_dim),
        )

        # Classifier head
        self.classifier = nn.Linear(feat_dim, num_classes)

    def forward(self, x):
        feat = self.pool(self.features(x)).flatten(1)  # (B, 1280)
        neck = self.dropout(self.bn_neck(feat))          # BN-Neck
        emb  = F.normalize(self.embed_head(neck), p=2, dim=1)  # L2 norm
        logits = self.classifier(neck)                   # (B, 196)
        return logits, emb

6 PKSampler & Batch Construction

Why Random Sampling Breaks Triplet Loss

Standard random sampling for a 196-class dataset with batch size 32 typically results in each batch containing mostly single-instance classes. With only one image per class per batch, there are no valid same-class pairs — the triplet loss has no positives to work with and produces zero useful gradient.

Rule of thumb: For batch-hard triplet mining to work, every batch must contain at least 2 images from the same class. In practice, K ≥ 4 is recommended for stable gradients.

P×K Sampling

The PKSampler (also called "class-balanced batch sampler") constructs each mini-batch by:

  1. Randomly sample P classes from the dataset (e.g., P=32)
  2. For each selected class, randomly sample K images (e.g., K=4)
  3. Resulting batch size = P × K = 128
Guaranteed triplets per batch
Valid triplets ≥ P · K · (K−1) · (P−1) · K = 32 · 4 · 3 · 31 · 4 = 47,616 triplets

Batch-hard mining selects only the hardest P·K = 128 of these — but the pool ensures informative hard triplets are almost always available.

PKSampler implementation 
from collections import defaultdict
import random
from torch.utils.data import Sampler

class PKSampler(Sampler):
    def __init__(self, labels, P=32, K=4):
        self.P, self.K = P, K
        self.index_by_class = defaultdict(list)
        for i, lbl in enumerate(labels):
            self.index_by_class[lbl].append(i)
        self.classes = [c for c, idxs in self.index_by_class.items() if len(idxs) >= K]

    def __iter__(self):
        while True:
            batch_classes = random.sample(self.classes, self.P)
            batch = []
            for c in batch_classes:
                batch.extend(random.sample(self.index_by_class[c], self.K))
            yield from batch

    def __len__(self):
        return len(self.classes) // self.P

Loss Weight Scheduling

Starting with a low β for the triplet term avoids the instability common in early triplet training (before embeddings are meaningful, hard triplets are essentially random). Ramping β linearly over the first 10 epochs gives the CE loss time to structure the feature space, after which triplet loss refines the geometry. 

# Linear β warmup over first `warmup_epochs` epochs
beta_schedule = lambda epoch: min(1.0, 0.5 + 0.5 * epoch / warmup_epochs)

8 Evaluation Metrics

Classification Metrics (from Classifier Head)

Top-1 Accuracy

The predicted class (argmax of logits) matches the ground-truth label. Primary metric for VMMR benchmarking.

Stanford Cars SOTA: ~94–95% (with EfficientNetV2 or ViT + strong augmentation)

Top-5 Accuracy

Ground truth is within the top-5 predicted classes. Useful for understanding whether the model is "close" even when wrong. Typically 98–99% for SOTA models.

Retrieval Metrics (from Embedding Head)

Recall@K (R@1, R@5)

For a query image, are any of the K nearest neighbors in the embedding space the same class? R@1 ≈ top-1 accuracy for retrieval; R@5 measures the quality of the embedding neighborhood.

mAP@R (Mean Average Precision)

For each query, compute AP over the ranked list of retrieved images of the same class. Average over all queries. More sensitive to the full ranking than R@K.

Embedding Space Visualization

t-SNE or UMAP applied to the 256-d test embeddings gives an intuitive view of the learned space. Good embeddings show tight, well-separated clusters per class. Adjacent clusters should be semantically related (e.g., same model, different year).
